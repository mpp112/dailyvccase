# 一、总体 Plan

## 1. 北极星目标

每个工作日发布 1 篇“迷你投委会备忘录”，并同步形成 4 份可复用资产：

- 结构化案例记录；
- 逐条 Claim–Evidence 证据表；
- 中国对标及融资历史；
- 可在 3/6/12 个月复查的假设与指标。

日更不是目的，建立可验证的投资判断框架才是目的。

## 2. 初始研究边界

第一阶段只选：

- 地区：美国、英国、欧洲、以色列、新加坡；
- 阶段：Seed 到 Series B；
- 行业：最多 2 个；
- 时效：优先过去 72 小时公开披露的融资；
- 中国对标：1 家主要对标，最多 2 家补充对标；
- 投资机构分析：重点分析最多 3 家核心机构。

连续完成 20 个案例后，才扩大行业或地区。

## 3. 四阶段建设路线

### 阶段 A：定义与黄金样本

交付：

- 确定行业、阶段、地区与文章模板；
- 人工完成 5–10 个黄金样本；
- 对每个事实建立来源与可信度标签；
- 固化“什么叫直接对标”和“什么叫完成”。

通过条件：不同人依据同一 SOP，对同一案例的关键事实与对标结论基本一致。

### 阶段 B：半自动采集与结构化

交付：

- RSS/API/手工 URL 采集器；
- 候选事件去重；
- 公司、轮次、金额、投资方实体归一化；
- Claim–Evidence 数据结构；
- Markdown 报告渲染。

通过条件：黄金样本可重复导入，关键字段不丢失，来源可回溯。

### 阶段 C：规则化 QA 与案例工作台

交付：

- 红线检查；
- 来源覆盖率、事实完整度、对标匹配度评分；
- 缺失、冲突和低可信度事实自动提示；
- 日报候选池与案例状态看板。

通过条件：不合格案例不能进入 `ready_to_publish` 状态。

### 阶段 D：稳定后再定时化

交付：

- 每日候选发现任务；
- 每周复盘任务；
- 3/6/12 个月案例追踪提醒；
- 失败告警与人工审批。

通过条件：连续 10 次运行无数据污染、无重复发布、无关键事实错误后，才增加自动化程度。

## 4. 日、周、月运行节奏

### 每日

1. 发现 5–15 个候选；
2. 评分后保留 1 个主案例、1 个备用案例；
3. 完成海外标的研究、投资人研究和中国对标；
4. 通过质量门；
5. 发布并归档证据；
6. 将关键假设加入追踪队列。

### 每周

- 汇总 5 个案例；
- 比较融资热点、投资人 thesis、共同假设和失败风险；
- 检查本周事实更正率；
- 抽查至少 1 篇文章的全部引用；
- 更新关键词和对标词典。

### 每月

- 输出行业 thesis 更新；
- 回看已发布案例是否出现新融资、客户、产品或风险事件；
- 统计来源质量、研究耗时和返工原因；
- 移除低价值数据源，补充高价值来源；
- 只把已经稳定的步骤升级为 Skill 或 Automation。

## 5. 人机分工

| 工作 | 系统/Codex | 人工 |
|---|---|---|
| 候选发现、去重 | 主做 | 抽查 |
| 事实抽取 | 主做 | 关键项复核 |
| 数据归一化 | 主做 | 处理冲突 |
| 投资逻辑初稿 | 辅助 | 最终判断 |
| 中国对标候选 | 辅助 | 确认是否可比 |
| 融资历史整理 | 主做 | 金额与机构复核 |
| 质量规则 | 自动 | 红线审批 |
| 发布 | 生成草稿 | 必须批准 |

## 6. 案例状态机

`discovered → shortlisted → researching → evidence_complete → qa_failed / qa_passed → ready_to_publish → published → monitoring`

任何状态变化必须记录时间、操作者和原因。`qa_failed` 不允许绕过。
